# DLUT Object-Oriented Programming

## 目标

帮助用户理解面向对象分析与设计（OOA/OOD）、完成 UML 建模、应用设计模式、进行代码重构、完成课程项目和复习。

## 工作流

1. 识别主题：OOA 分析、OOD 设计、UML 建模、设计模式、代码重构或项目实训。
2. 课程索引读取 `references/course-index.md`。
3. 开发工具和环境读取 `references/tooling.md`。
4. 设计模式/UML 速查读取 `references/cheatsheet.md`。
5. 复习要点读取 `references/review.md`。

## 质量规则

- 面向对象分析要先识别参与者、用例和领域对象，再建立类图。
- 设计模式应用要明确问题上下文、解决方案结构和权衡。
- UML 建模要区分分析类图（领域模型）和设计类图（实现细节）。
- 代码重构要遵循 SOLID 原则，每次只做一个小步重构并验证。
- 项目文档要包含需求规格、设计说明、类图、时序图和测试计划。

# 面向对象方法程序设计及实训复习要点

## 核心概念对比
| 概念 | 要点 |
|------|------|
| OOA vs OOD | OOA 面向问题域，建立领域模型；OOD 面向解域，建立设计模型 |
| 聚合 vs 组合 | 聚合部分可独立存在；组合部分随整体生命周期结束 |
| 继承 vs 组合 | 继承是 is-a，紧耦合；组合是 has-a，更灵活 |

## 易错点
- 关联和依赖的区别：关联是结构关系，依赖是使用关系
- 里氏替换要求子类不能弱化父类的前置条件，不能强化后置条件
- 工厂方法 vs 抽象工厂：前者一个工厂创建一个产品，后者一个工厂创建一族产品
- 观察者模式中，通知顺序不应有依赖

## 大题常见类型
1. 用例图与用例描述
2. 类图设计（含关系识别）
3. 时序图/活动图绘制
4. 设计模式应用场景与类图
5. 代码重构分析与改进方案
6. 项目设计文档撰写

# 面向对象方法程序设计及实训速查

## UML 关系符号速查
| 关系 | 符号 | 含义 |
|------|------|------|
| 关联 | ——→ | 一个类知道另一个类 |
| 聚合 | ◇—— | 整体-部分，部分可独立存在 |
| 组合 | ◆—— | 整体-部分，部分随整体销毁 |
| 继承 | △—— | is-a 关系 |
| 依赖 | - -→ | 使用关系 |
| 实现 | △- - | 接口实现 |

## SOLID 原则速查
| 原则 | 含义 |
|------|------|
| S - 单一职责 | 一个类只负责一件事 |
| O - 开闭原则 | 对扩展开放，对修改关闭 |
| L - 里氏替换 | 子类可替换父类而不影响程序 |
| I - 接口隔离 | 客户端不应依赖不需要的接口 |
| D - 依赖倒置 | 依赖抽象，不依赖具体实现 |

## 常用设计模式速查
| 模式 | 类型 | 场景 |
|------|------|------|
| 单例 | 创建型 | 全局唯一实例 |
| 工厂方法 | 创建型 | 创建对象时解耦 |
| 观察者 | 行为型 | 事件订阅/通知 |
| 策略 | 行为型 | 算法族互相替换 |
| 装饰器 | 结构型 | 动态添加功能 |
| 适配器 | 结构型 | 接口不兼容的转换 |
| 模板方法 | 行为型 | 算法骨架，子类实现步骤 |

## 代码坏味道速查
- 重复代码 → 提取方法/类
- 过长方法 → 提取方法
- 过大类 → 提取子类/接口
- 过长参数列表 → 引入参数对象
- 发散式变化 → 提取类
- 霰弹式修改 → 移动方法/字段

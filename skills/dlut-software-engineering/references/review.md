# 软件工程复习要点

## 核心概念对比
| 概念 | 要点 |
|------|------|
| 瀑布 vs 敏捷 | 文档驱动/阶段分明 vs 迭代交付/响应变化 |
| 黑盒 vs 白盒 | 不关心内部结构 vs 基于代码结构 |
| 耦合 vs 内聚 | 模块间依赖度 vs 模块内相关度；低耦合高内聚 |

## 易错点
- UML 中关联、聚合、组合的区别在菱形形状和生命周期
- 工厂方法 vs 抽象工厂：前者一个工厂一个产品，后者一族产品
- 路径覆盖最强但成本最高；语句覆盖最弱
- 功能点估算与代码行数无关，基于用户需求

## 大题常见类型
1. 用例图与用例描述编写
2. 类图与时序图绘制
3. 设计模式应用场景分析与类图
4. 测试用例设计（等价类、边界值）
5. 项目进度网络图与关键路径计算
6. 软件过程模型对比与选择

# DLUT Software Engineering

## 目标

帮助用户理解软件工程方法论、完成 UML 建模、应用设计模式、编写测试用例、管理软件项目文档和复习。

## 工作流

1. 识别主题：需求、设计、建模、实现、测试、维护、管理或文档。
2. 课程索引读取 `references/course-index.md`。
3. 工具链读取 `references/tooling.md`。
4. 模式/方法速查读取 `references/cheatsheet.md`。
5. 复习要点读取 `references/review.md`。

## 质量规则

- UML 图要区分静态结构（类图、组件图）和动态行为（时序图、活动图、状态图）。
- 设计模式要说明适用场景、类图结构和 Java/Python 伪代码示例。
- 测试用例要覆盖等价类划分、边界值、路径覆盖等策略。
- 项目文档要结构清晰，包含需求规格、设计说明、测试计划和用户手册。

# DLUT Programming Practice

## 目标

帮助用户完成程设实践项目理解、EasyX 图形代码查错、交互逻辑设计、游戏循环调试和报告整理。

## 工作流

1. 判断项目是 Clock、Canvas、Pinball、Miner2D、Miner 或新项目。
2. 项目索引读取 `references/course-index.md`。
3. EasyX/Visual Studio 配置读取 `references/tooling.md`。
4. 图形和游戏循环速查读取 `references/cheatsheet.md`。
5. 报告和复习读取 `references/review.md`。

## 质量规则

- 尊重 EasyX 和课程现有工程结构，优先最小修改。
- 检查消息循环、双缓冲、帧率控制、资源路径和边界碰撞。
- 不替用户提交完整项目；帮助解释、查错和规划。

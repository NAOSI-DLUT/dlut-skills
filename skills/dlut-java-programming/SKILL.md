# DLUT Java Programming

## 目标

帮助用户理解 Java 语法与面向对象思想、完成课程实验、检查代码质量、调试异常和复习。

## 工作流

1. 识别主题：语法基础、OOP、集合、异常、IO、多线程、网络、数据库或 GUI。
2. 课程索引读取 `references/course-index.md`。
3. 开发环境读取 `references/tooling.md`。
4. API/语法速查读取 `references/cheatsheet.md`。
5. 复习要点读取 `references/review.md`。

## 质量规则

- 面向对象设计要检查封装、继承、多态的合理使用，避免过度继承。
- 集合选择要根据场景对比 List/Set/Map 的实现类和性能特征。
- 多线程代码要检查同步、死锁、线程安全和资源释放。
- 异常处理要区分受检/非受检异常，避免空 catch 块和过度捕获。
